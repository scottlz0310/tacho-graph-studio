# tacho-graph-studio 要件定義書

- 作成日: 2026-07-17 / 最終更新: 2026-07-18
- ステータス: ドラフト（新規リポジトリ作成までの暫定配置。新 repo 作成後に移管する）
- 対象: TachoGraphWizard（GIMP 3 プラグイン）の後継となる Windows デスクトップアプリ

---

## 1. 背景と目的

### 1.1 現行の課題（TachoGraphWizard / GIMP 3 プラグイン）

- GIMP 3 のインストール・バージョン追従が前提となり、セットアップ・運用のハードルが高い
- GIMP プラグインの UI フレームワーク（GimpUi/GTK）では、名簿連携・単一画面での逐次処理といった UX を実現するのに限界がある
- 名簿（車両・運転者）は CSV の手動管理であり、machinery-report-system で運用中の Supabase 名簿と二重管理になっている

### 1.2 目的

タコグラフチャート画像処理ワークフローを、GIMP 非依存のスタンドアロン Windows デスクトップアプリ **TachoGraphStudio** としてリライトする。

- スキャン画像の取り込みから透過 PNG 保存までを単一アプリ・単一画面で完結させる
- 名簿を machinery-report-system の Supabase から直接取得し、二重管理を解消する
- UI モック `tacho-graph-studio.html`（Dropbox/工事データ/TachoGraphStudio）の構成を実装の基準とする

### 1.3 決定事項

| 項目 | 決定 | 理由 |
|---|---|---|
| リポジトリ戦略 | **新規リポジトリ `tacho-graph-studio`**（ケバブケース） | 技術スタックが完全に別物（Python/GIMP → C#/.NET/WinUI3）のため、CI・Renovate・リリースを分離する |
| UI フレームワーク | **WinUI 3**（Windows App SDK） | モダンな Windows ネイティブ UI。モックも WinUI 3 デザインを前提に作成済み |
| Supabase アクセス | **anon キーによる読み取り専用** | `machine_picklist` ビューは anon に SELECT 付与済み。名簿読み取りのみでありログイン UI を不要化できる |
| anon キーの受け渡し | **初回起動時に入力し、暗号化してローカル永続化**（FR-23） | 名簿に個人名を含むため repo・配布物には埋め込まない。設定画面で変更可能 |
| ランタイム | **.NET 10 (LTS)** | 2026 年時点の現行 LTS。新規開発で旧バージョンを選ぶ理由がない |
| GIMP プラグイン版の扱い | **凍結（メンテのみ）** | 新アプリが実運用に乗るまで現状維持。以降は機能追加せずバグ修正のみとし、最終的に README に移行案内を記載してアーカイブ |
| 画像処理ライブラリ | **OpenCvSharp を第一選択** | 円検出・楕円フィット・しきい値処理の実績が豊富。不適合が判明した場合のみ代替（SkiaSharp/Win2D）を再検討 |
| 配布形態 | **MSIX + `.appinstaller`**（Web/Storage 配置） | 自動更新に対応。自己署名証明書を使用し、証明書インポートを含む導入手順/スクリプトを提供する |
| タコ対象フラグ | **`machines.is_tacho_target` (boolean)** | `machine_picklist` に公開。フラグ付与 UI は machinery-report-system のタスクペインを拡張して実装する |
| テンプレート | **現行 JSON 定義を継承候補とし、GUI 編集 UI を必須とする** | 継承で不足があれば再設計し、その場合は旧テンプレートファイルのインポーターを設ける |
| `.appinstaller` 配置先 | **GitHub Releases**（2026-07-18 決定） | 追加インフラ不要。`releases/latest/download/…` の固定 URL が得られ自動更新に使える |

## 3. 機能要件

### 3.1 入力

- **FR-01**: A3 スキャン画像（PDF または JPEG）を読み込めること。1 シートに最大 6 枚の円盤を含む
- **FR-02**: 複数ページ PDF・複数ファイルの連続処理（バッチ投入）ができること

### 3.2 画像処理パイプライン

- **FR-03**: シート画像から円盤ごとの領域を自動検出・分割できること（しきい値・パディング調整可能）
- **FR-04**: 分割結果をサムネイルナビゲーション（No.1〜No.6）で一覧・選択できること
- **FR-05**: 円盤の背景（白地）を自動除去し、アルファチャンネル化できること（楕円フィット・パディング調整可能）
- **FR-06**: プレビュー上に画像の回転とは独立した十字ガイドを表示し、回転補正の基準にできること。十字ガイドは画像と同じコンテンツ座標系でズーム・パンに追従すること
- **FR-07**: 回転補正をスライダー（-180°〜+180°）・数値入力・リセットで行えること。プレビューはリアルタイム追従
- **FR-08**: プレビューのズームイン/アウト・全画面拡大ができること

### 3.3 名簿連携（Supabase）

- **FR-09**: machinery-report-system の Supabase から `machine_picklist` ビューを anon キーで読み取り、名簿（管理番号・機械名・登録番号・運転者等）を表示できること。**書き込みは一切行わない**
- **FR-10**: 名簿はローカルにキャッシュし、オフライン時（現場等）は最終取得分で動作できること
- **FR-11**: フィルタ機能は machinery-report-system の `MachineSelectDialog`（office-js 実装）の仕組みを移植すること
  - シーズンフィルタ（`work_period`: 冬期(除雪) / 夏期維持 / 通年 / 全て）
  - キーワード検索・管理番号ジャンプ
  - フィルタ状態のローカル永続化
- **FR-12**: **タコチャート対象フラグ（新設）**: タコチャートを利用する機械は限られるため、`machines` テーブルに `is_tacho_target boolean` 列を追加し、`machine_picklist` ビューに公開する。tacho-graph-studio の名簿はこのフラグでデフォルト絞り込みを行う
  - スキーマ変更のマイグレーション SQL（up/down ペア）は **machinery-report-system リポジトリ側**で管理する
  - フラグの付与・編集 UI は **machinery-report-system のタスクペイン（Office Add-in）を拡張**して実装する。本アプリでは行わない

### 3.4 文字入れ・メタデータ

- **FR-13**: 名簿の行選択で登録番号・運転者名が入力欄に自動反映されること
- **FR-14**: 処理対象日をトップバーで一括指定でき、印字日付に同期されること
- **FR-15**: 印字日付・登録番号・運転者名は確定前に手修正できること（臨時運転者対応）
- **FR-16**: チャート紙様式（テンプレート）を選択でき、様式ごとに文字入れ位置が切り替わること（例: ヤザキ標準=中央上、タスクメーター用=右下寄り）
- **FR-17**: 「手書きのため自動文字入れをスキップ」を円盤ごとに選択でき、その場合は文字入れを行わず、ファイル名を `YYYYMMDD_登録番号_運転者_手書き.png` 形式とすること。トップバーから現在の設定を全円盤へ一括適用できること
- **FR-18**: 文字入れ内容はプレビュー上にリアルタイム表示されること（画像回転とは独立したレイヤー）

### 3.5 保存

- **FR-19**: アルファチャンネル付き PNG でローカル保存できること
- **FR-20**: ファイル名は通常 `YYYYMMDD_登録番号_運転者.png`、手書きスキップ時は FR-17 の形式とし、保存前にプレビュー表示されること
- **FR-21**: 「確定保存して次へ」で保存と同時に次の円盤（サムネイル）へ遷移し、逐次処理が途切れないこと

### 3.6 設定・永続化

- **FR-22**: 出力先フォルダ・前回日付・テンプレート選択・フィルタ状態・ウィンドウ状態を永続化し、次回起動時に復元すること
- **FR-23**: Supabase の URL・anon キーは**初回起動時にユーザーが入力**し、ローカルに永続化する（設定画面から変更・再入力可能）
  - 保存時は Windows の資格情報保護（`PasswordVault` または DPAPI）で暗号化し、平文の設定ファイルには書かない
  - リポジトリ・ソースコード・MSIX パッケージには埋め込まない。anon キーは設計上クライアント公開前提の値だが、`machine_picklist` に個人名（運転者）が含まれるため、公開 repo・配布物経由での漏洩経路を作らない
  - 未設定またはキー無効時は名簿パネルに設定導線を表示し、名簿以外の機能（画像処理・手入力での文字入れ・保存）は利用可能とする

### 3.7 テンプレート管理

- **FR-24**: チャート紙様式テンプレート（文字入れ位置・フォント・色等の定義）を **GUI で編集**できること。プレビュー上で位置を確認しながら調整できることが望ましい
- **FR-25**: テンプレート定義は現行 GIMP 版の JSON フォーマットを継承候補とし、そのまま使える場合は互換維持する
- **FR-26**: フォーマットを再設計する場合は、旧テンプレートファイル（GIMP 版 JSON）のインポーターを設けること

## 4. UI 要件

UI モック `tacho-graph-studio.html` を正とする。単一画面（ウィザード遷移なし）で以下を構成する。

```
┌─────────────────────────────────────────────────┐
│ トップバー: 処理対象日 / 手書きスキップ                 │
├──────────────┬──────────────────────────────────┤
│ 名簿サイドバー   │ ステージ                           │
│  - フィルタ      │  - コマンドバー (取込/様式/設定)       │
│  - 検索         │  - サムネイル縦列 | 円盤プレビュー     │
│  - 名簿グリッド   │                  | 回転縦レール      │
│                │    (十字ガイド/ズーム/角度入力)       │
│                │  - メタデータエディタ (日付/番号/運転者) │
│                │  - 保存先プレビュー / 確定保存して次へ   │
└──────────────┴──────────────────────────────────┘
```

- **UR-01**: Fluent Design（WinUI 3 標準）に準拠し、ライト/ダークテーマに追従すること
- **UR-02**: 名簿グリッドは行クリックで選択、選択行のハイライト表示
- **UR-03**: 処理済み・未処理・スキップのステータスがサムネイルで判別できること
- **UR-04**: サムネイルはプレビュー左側へ縦配置し、表示領域を超える場合は垂直スクロールで選択できること。回転補正はプレビュー右側の縦レールへ配置すること

## 5. 非機能要件

- **NFR-01**: 対応 OS は Windows 11（Windows App SDK の要求に従う）
- **NFR-02**: GIMP・Python 等の外部ランタイムに依存しないこと（MSIX パッケージに必要なランタイムを同梱）
- **NFR-02b**: 配布は **MSIX + `.appinstaller`** とし、Web または Storage 上に配置して自動更新に対応すること。自己署名証明書で署名し、証明書のインポートを含むセットアップ手順・スクリプトを提供すること
- **NFR-03**: A3 600dpi 級のスキャン画像でも分割〜プレビューが実用速度（数秒以内）で動作すること
- **NFR-04**: Supabase 接続不可時もオフラインキャッシュで主要ワークフローが継続できること（FR-10）
- **NFR-05**: 名簿・出力ファイルに含まれる個人名等はログに出力しないこと
- **NFR-06**: ライセンスは MIT（現行踏襲）。ただしサンプル画像は匿名・低解像度化して公開すること

## 6. 技術スタック

| 領域 | 選定 | 備考 |
|---|---|---|
| UI | WinUI 3 + Windows App SDK（確定） | MSIX packaged |
| 言語/FW | C# / **.NET 10 (LTS)** + CommunityToolkit.Mvvm | |
| 画像処理 | **OpenCvSharp（第一選択）** | 円検出・楕円フィット・しきい値処理。不適合時のみ SkiaSharp/Win2D を再検討 |

| 品質 | dotnet format / .editorconfig / Roslyn analyzers | ユーザー標準ツールチェーンに準拠 |
| 依存更新 | Renovate（`github>scottlz0310/renovate-config` を拡張） | |
